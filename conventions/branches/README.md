# Branch Naming Conventions

To keep our repository organized and predictable, please follow these branch naming conventions. 

## Format
`[type]/[ticket-id]-[short-description]`

## Types

- `feat/`: For new features or significant additions to the codebase.
- `fix/`: For bug fixes and patches.
- `chore/`: For routine tasks, maintenance, dependency updates, or configuration changes.
- `refactor/`: For code refactoring without changing observable behavior.
- `docs/`: For documentation updates.
- `test/`: For adding or updating tests.

## Examples

- `feat/PROJ-123-matrix-table-reordering`
- `fix/PROJ-456-resolve-import-conflict`
- `chore/PROJ-789-update-dependencies`
- `refactor/PROJ-101-extract-form-renderer`

## Best Practices

1. Use lowercase letters and separate words with hyphens (`-`).
2. Keep the description concise but descriptive.
3. Always include the ticket ID if applicable.
